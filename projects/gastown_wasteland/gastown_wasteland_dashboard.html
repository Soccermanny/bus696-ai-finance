What is this? This interactive dashboard demonstrates how two new ideas from software engineer Steve Yegge — Gas Town and The Wasteland — connect directly to the finance and ML concepts we've been learning all semester.

The Big Idea

In this class, we've built ML models to predict markets, used walk-forward validation to avoid overfitting, studied how hedge funds blow up when they overbet, and learned the Kelly Criterion for sizing positions based on your edge. We've also seen how market regimes break models trained in a single environment.

Gas Town and The Wasteland are frameworks for scaling AI agent development — running dozens or thousands of AI coding agents in parallel. But the deeper insight is that the same principles that govern smart trading also govern smart trust in AI systems:

  • Kelly Criterion says: don't overbet a small edge, or you go bankrupt. The Wasteland says: don't over-trust an unproven agent, or your system breaks.
  • Walk-forward validation says: test on unseen data, not training data. Wasteland stamps are earned on out-of-sample performance, not promises.
  • Market regimes mean models fail when conditions change. Trust levels get demoted when agents fail in new environments — just like a quant strategy gets pulled in a drawdown.
  • Hedge fund blow-ups (LTCM, Amaranth, Archegos) happened because of overleveraging. In the Wasteland, promoting an agent to L3 Maintainer too fast is the same mistake — giving it too much influence before it's proven.

What Would It Take to Run Real Gas Town on This?

If we wanted to use actual Gas Town to build this trading system, here's what that would look like:

  1. Install Gas Town — Yegge's orchestration layer that manages multiple Claude Code instances from a single workspace. You'd set up a Town for the project.
  2. Define Rigs — Each Rig gets its own directory and task scope. Rig 1 might be /signals/momentum/, Rig 2 /signals/mean_reversion/, Rig 3 /regime_detection/, Rig 4 /integration/.
  3. Spin up Polecats — Each Rig launches 2–5 Claude Code instances. Polecat 1 in Rig 1 writes the feature engineering code. Polecat 2 writes the model training code. Polecat 3 writes unit tests. They all write real Python files simultaneously.
  4. The Mayor reviews — As Polecats finish, the Mayor agent checks that interfaces match (does the momentum signal output match what the integration rig expects?), resolves merge conflicts, and runs integration tests.
  5. What you'd need:
    • A Claude API key or Claude Pro/Max subscription (to run multiple Claude Code instances)
    • Gas Town installed (npm install -g gastown or similar — check Yegge's repo for current instructions)
    • A well-defined CLAUDE.md project file so each Polecat understands the codebase conventions
    • Clear task decomposition — each Molecule (task) needs to be small enough for one agent to complete independently
  6. For the Wasteland layer — you'd additionally need a DoltHub database, the wl CLI, and other Gas Towns to federate with. This is where teams at different universities or firms could share validated components (a walk-forward backtest framework built at Oxford becomes available to everyone).
The bottom line: Real Gas Town would let you build this entire system in a day or two instead of writing it by hand — with 20+ AI agents writing code in parallel, each specializing in one piece. What we've built here is a teaching tool that uses the same conceptual architecture (specialized workers, trust-weighted integration, earned reputation) to demonstrate how these ideas connect to Kelly sizing, walk-forward validation, and the other finance topics from this class.

How This Simulation Was Built

The underlying simulation uses real SPY data (2010–2026) downloaded via yfinance. Four ML models — two logistic regressions and two random forests from scikit-learn — each trained on different feature subsets (momentum, mean-reversion, volatility regime, and full-feature). The backtest uses annual walk-forward retraining with an expanding window: train on all prior years, test on the next year, retrain, repeat. No lookahead bias, no random splits — exactly the methodology we've emphasized all semester.

What Are Gas Town and The Wasteland?

Two paradigms from Steve Yegge (2026) for scaling AI-assisted development from one agent to thousands.

⛽ Gas Town

An AI agent orchestration system — an "IDE" for running 20-30 concurrent AI coding agents on a single project.

Town= your workspace
Rigs= individual project components
Mayor= supervisor agent that coordinates
Polecats= worker agents that write code
Molecules= workflow task definitions
Key insight: Instead of 1 developer + 1 AI, run dozens of agents in parallel with a supervision hierarchy. It's Erlang-style supervisor trees applied to AI coding.

🌎 The Wasteland

A federated trust network connecting thousands of Gas Towns via decentralized reputation, built on DoltHub.

Wanted Board= shared work items anyone can claim
Stamps= GPG-signed reputation tokens (quality 1-5)
Trust Levels= L1 registered → L2 contributor → L3 maintainer
Federation= sovereign forks of a shared commons DB
Key insight: Reputation is portable. Stamps follow your identity across Wastelands. Trust is earned, not assigned — just like Kelly says position size should be earned by proven edge.

The Kelly-Wasteland Parallel

Both Kelly and the Wasteland answer the same question: "How much should I commit given uncertainty?"

Kelly CriterionWasteland Trust
How much to bet?↔How much to trust?
Edge / Variance↔Avg Stamps / Variability
Full Kelly = aggressive↔L3 Maintainer = full trust
Half Kelly = conservative↔L2 Contributor = partial trust
Overbet → RUIN↔Over-trust → SYSTEM FAILURE
"We're Nobel laureates, lever up!"
— LTCM, 1998		 ↔ "This agent passed 2 tests, give it L3!"
— Reckless Wasteland operator
Kelly investors look boring until rivals blow up ↔ Careful validation looks slow until fast-trusters ship bugs
The Wasteland IS Kelly for trust. You don't give a new rig full maintainer access just because it did one good thing — that's overbetting. You gradually increase trust as evidence accumulates, the same way Kelly gradually increases position size as edge is confirmed.

Watch It in Our Data

Notice how agents get demoted after bad years (2015, 2022). The system doesn't just accumulate trust — it also withdraws it, just like Kelly shrinks position size when edge fades.

Discussion Questions

  1. Numerai is already a proto-Wasteland for finance. Thousands of data scientists submit predictions, earn reputation. How does Gas Town / Wasteland compare?
  2. If you had to design trust for a quant Wasteland, how would you handle regime changes? An L3 agent from a bull market might fail in a crisis.
  3. If the best strategy gets shared across 1,000 Gas Towns, what happens to the alpha? Is this Grossman-Stiglitz in action?
  4. Gas Town builds systems in days vs months. But LTCM was also built fast by geniuses. How do you balance speed with Kelly-style caution?
